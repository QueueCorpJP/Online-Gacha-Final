✅ 修正ステップ：
① prize.entity.ts が存在するか確認
bash
コピーする
編集する
ls src/entities/prize.entity.ts
もしファイルがないなら、作成する必要があります。

② Prize エンティティを User に適切に import しているか確認
ts
コピーする
編集する
// user.entity.ts
import { Prize } from './prize.entity'; // 相対パス注意
③ AppDataSource（または TypeOrmModule.forRoot({...})） に Prize が含まれているか
ts
コピーする
編集する
TypeOrmModule.forRoot({
  // ...
  entities: [User, Prize, ...],
})
または entities: [__dirname + '/**/*.entity{.ts,.js}'] のようにワイルドカードでカバーしているか。

④ Prize エンティティに逆側のリレーションがあるかも確認
ts
コピーする
編集する
// prize.entity.ts
@ManyToOne(() => User, user => user.prizes)
user: User;
🧪 もしエンティティが多いなら：
ルートで以下を使って、TypeORM が読み取っている .entity.ts の一覧を洗い出せます：

bash
コピーする
編集する
grep -r '@Entity' src/entities
対応後、サーバー再起動：

bash
コピーする
編集する
pm2 restart gacha-backend
